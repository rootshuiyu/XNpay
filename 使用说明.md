# 犀牛支付 - 使用说明

## 一、你看到的「收银台地址」是什么？

- **收银台地址**（系统管理 → 收银台）是**可选功能**：用来配置「固定入口」的收银台链接（如 `/pay/xxx`），并关联某个游戏账号。
- **日常支付流程不需要先在这里添加**。用户付款时打开的页面是 **`/cashier/订单号`**，由下单接口返回的 `cashier_url` 决定，和本页的「添加收银台」无关。

---

## 二、推荐使用流程（从零到收款）

### 1. 配置游戏通道与账号

1. **游戏通道**：左侧「游戏通道」→ 添加通道（名称、通道编码 `channel_code`、支付类型选支付宝等）。
2. **游戏账号**：左侧「游戏账号」→ 添加账号 或 批量导入。账号即畅游等平台的通行证（账号/密码），用于上号器自动登录、下单、取码。
3. 通道的 **通道编码** 后面商户下单时要传，务必记下（如 `game001`）。

### 2. 上号器（自动取码、轮询）

1. 左侧进入 **「上号器管理」**。
2. 点击 **「启动 Bot」**，让服务开始处理队列中的订单（自动登录 → 下单 → 取二维码 → 轮询支付状态）。
3. （可选）**代理池**：若要反检测，可在「代理池」里添加 HTTP/SOCKS5 代理并启用。
4. 订单列表里可查看每条订单的 Bot 状态（排队中 / 处理中 / 二维码就绪 / 轮询中 / 已完成 / 失败），失败可点「重试」。

### 3. 商户对接（对方系统调用你）

商户侧需要：

- **支付域名**：你部署的地址，如 `http://137.220.221.215`
- **密钥**：在游戏通道里配置（如通道的扩展配置/密钥），用于签名。
- 调用 **下单接口** `POST /pay/create`，传 `channel_code`、`out_trade_no`、`amount`、`notify_url`、`sign` 等（见「API 文档」）。
- 接口返回里的 **`cashier_url`**（如 `/cashier/XN20260312150405123456`），商户让用户打开：  
  **`你的域名 + cashier_url`**，即用户看到的收银台页面。

### 4. 用户付款时会发生什么

1. 用户打开 `https://你的域名/cashier/订单号`。
2. 若上号器已取到码：页面显示**支付宝二维码**，用户扫码付款；后台自动轮询，支付成功后自动回调商户 `notify_url`。
3. 若未启用上号器或取码失败：页面会显示**游戏账号信息**，用户按说明手动登录游戏充值后，点「我已完成充值」确认。

## 三、功能对应关系（简表）

| 你想做的事           | 在哪里操作 |
|----------------------|------------|
| 商户能下单、选通道   | 游戏通道 → 通道编码给商户；API 文档里看参数 |
| 自动出二维码、自动轮询 | 上号器管理 → 启动 Bot；保证有可用游戏账号 |
| 多 IP/反检测         | 上号器管理 → 代理池 → 添加代理并启用 |
| 查看/重试 Bot 订单   | 上号器管理 → 订单处理 |
| 固定收银台链接       | 收银台地址 → 添加收银台（可选） |

---

## 四、快速自测

1. 在「游戏通道」建一个通道，例如编码 `test`。
2. 在「游戏账号」下为该通道添加至少一个畅游账号。
3. 「上号器管理」→ 启动 Bot。
4. 用 API 文档里的方式调一次 `POST /pay/create`（或用 Postman），拿到 `cashier_url`。
5. 浏览器打开 `http://137.220.221.215/cashier/返回的订单号`，应看到金额、二维码或账号信息。

按以上顺序即可完成「接下来怎么使用」的完整流程；**不需要先在「收银台地址」里添加数据**也能正常收款。
